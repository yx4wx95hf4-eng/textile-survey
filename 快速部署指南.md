## 📋 方案一：GitHub Pages 部署（详细步骤）

### 第一步：准备 GitHub 仓库

1. 访问 [github.com](https://github.com)，登录您的账号
2. 点击右上角 `+` → `New repository`
3. 填写仓库信息：
   - Repository name: `textile-survey`（可以自定义）
   - 选择 `Public`
   - **不要**勾选 "Add a README file"
4. 点击 `Create repository`

### 第二步：推送代码到 GitHub

**方法 A：使用 Git 命令（推荐）**
```bash
# 打开 Git Bash，运行以下命令（替换 YOUR_USERNAME 为您的 GitHub 用户名）
cd /c/Users/wisdo/WorkBuddy/2026-06-22-16-16-28
git remote add origin https://github.com/YOUR_USERNAME/textile-survey.git
git branch -M main
git push -u origin main
# 输入您的 GitHub 用户名和密码（或 Personal Access Token）
```

**方法 B：直接上传文件（简单）**
1. 在 GitHub 仓库页面，点击 `Add file` → `Upload files`
2. 上传以下文件：
   - `outputs/frontend/survey.html`
   - `outputs/frontend/admin.html`
   - `outputs/frontend/config.js`
3. 点击 `Commit changes`

### 第三步：启用 GitHub Pages

1. 在 GitHub 仓库页面，点击 `Settings`
2. 左侧菜单点击 `Pages`
3. Source 选择 `main` 分支
4. 点击 `Save`
5. 等待 1-2 分钟，您将获得网址：
   - 问卷：`https://YOUR_USERNAME.github.io/textile-survey/outputs/frontend/survey.html`
   - 后台：`https://YOUR_USERNAME.github.io/textile-survey/outputs/frontend/admin.html`

## 📋 方案二：Google Apps Script 后端部署

### 第一步：创建 Google Apps Script

1. 访问 [script.google.com](https://script.google.com)
2. 点击 `新建项目`
3. 将项目名称改为 `纺织问卷后端`
4. 删除默认代码，粘贴 `outputs/backend/gas_backend.js` 的内容
5. 点击 `保存`

### 第二步：创建关联的 Google Sheet

1. 在 Google Apps Script 编辑器中，点击 `运行` → `run function` → `getSheet`
2. 首次运行需要授权，点击 `查看权限` → 选择您的 Google 账号 → 点击 `高级` → `转到...（不安全）` → `允许`
3. 运行完成后，打开 [drive.google.com](https://drive.google.com)，您会看到一个名为 `纺织问卷数据` 的 Google Sheet

### 第三步：部署为 Web App

1. 点击 `部署` → `新建部署`
2. 选择部署类型：`Web 应用`
3. 配置：
   - 说明：填写 `v1`
   - 执行身份：`我（您的邮箱）`
   - 访问权限：`任何人`
4. 点击 `部署`
5. **复制生成的 Web App URL**（类似 `https://script.google.com/macros/s/XXXXXX/exec`）

### 第四步：修改前端配置

1. 打开 `outputs/frontend/config.js`
2. 将 `YOUR_SCRIPT_ID` 替换为您的实际 Script ID（URL 中的 `XXXXXX` 部分）
3. 保存文件
4. 重新推送到 GitHub（或重新上传文件）

## 🆘 常见问题

**Q：GitHub Pages 网址访问不了？**
A：检查 `Settings` → `Pages` 中的状态，确保显示 `Your site is published at ...`

**Q：提交问卷后数据没有保存？**
A：检查 `config.js` 中的 `API_BASE` 是否正确，以及 Google Apps Script 是否已部署。

**Q：微信中打开网址显示“无法访问”？**
A：微信可能会拦截 GitHub Pages 网址，建议使用**短链接服务**（如 bit.ly）或申请微信官方小程序。
